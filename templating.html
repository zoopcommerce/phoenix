Templating Markup

Custom templates for your Zoop online store

Introduction

To allow simple and complete storefront customization we are utilizing Twig, which is a robust templating engine for PHP.

Twig allows you to add conditional statements, loops and of course, variable substitution.

The semantics will be familiar as many other popular e-commerce platforms, including Shopify, use similar markup in their templates. And even if you have never used a templating engine before, it is simple enough to get up and running quickly!

Below is a break down of some of the concepts you will need to know to begin templating.

Accessing variables

The fundamental aspect of a templating engine is to allow you access to dynamic information (eg product name, price etc), and present that in your page.

To do so is quite simple:

<span>Store name: </span><span>{{ store.name }}</span>

As you can see, by enclosing a variable in double moustaches {{ ... }} it will attempt to parse the variable.

Control Structures

A control structure refers to all things that dictate the flow of a program eg. conditions such as if/elseif/else, for-loops, as well as things like blocks. Control structures appear inside {% ... %} blocks.

Conditions

The following example checks the product to see if it in fact has a sale price. If it does, it will display both the sale price and the regular price for comparison. Otherwise it just displays the regular price. 

{% if(product.price.sale > 0) %}
    <span class="on-sale-price">On Sale now: ${{ product.price.sale }}</span>
    <span class="regular-price">{{ product.price.full }}</span>
{% else %}
    ${{ product.price.full }}
{% endif %}

Loops

This example iterates through a product image set and displays each thumbnail.

{% for image in product.images %}
    <img src="{{ image.thumbnail.src }}" alt="{{ image.thumbnail.alt }}" width="{{ image.thumbnail.width }}" height="{{ image.thumbnail.height }}">
{% endfor %}

Includes

You are able to include other templates to allow your project to be more modular and reusable. For instance, you can include a navigation template easily on each page just by:

{% include "navigation.html" %}

Further reading

There's a load more functionality such as functions, filters, blocks and much more! Twig has some great documentation that can be found by visiting: http://twig.sensiolabs.org/documentation

Global variables

We have implemented a number of convenient global variables which can be accessed from all pages.

Site

The site global variable contains information pertaining to the site:

var site = {
    css: array | [string], // contains an array of css files. This only applies if you are using Zoop default templates.
    dojo: array | [string], // contains an array of dojo javascript files. This only applies if you are using Zoop default templates.
    head: {
        favicon: string | null, // favourite icon
        meta: {
            description: string // The page meta description
        },
        openGraph: string, // This contains open graph information https://developers.facebook.com/docs/opengraph/
        title: string // The page title which is made up from product, category and site name
    },
    javascript: array | [string], // contains an array of javascript files. This only applies if you are using Zoop default templates.
    status: {
        active: boolean, // Is the store active or not
        message: string | null // Contains a message which can be displayed when a store is inactive
    },
    url: {
        checkout: string, // The absolute url of the checkoute
        currentPage: {
            absolute: string, // The absolute url of the current page
            relative: string // The relative url of the current page
        },
        home: string // The absolute url of the home page
    }
}

To access these variables just use "dot" notation eg.

{{ site.url.currentPage.absolute }}
